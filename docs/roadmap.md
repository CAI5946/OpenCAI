# OpenCAI Roadmap

## 核心问题

OpenCAI 如何把一个自然语言开发请求，转化为可验证的代码修改循环？

## 第一阶段：核心循环

目标：拆清楚一次任务从输入到完成的最小闭环。

观察点：

- 如何理解用户目标
- 如何选择需要读取的上下文
- 如何决定使用哪些工具
- 如何修改文件
- 如何运行验证
- 如何在失败后继续迭代

## 第二阶段：上下文系统

目标：实现 OpenCAI 对项目规则、文件内容、历史消息和工具结果的最小利用方式。

观察点：

- 入口规则如何影响行为
- 它优先读哪些文件
- 它如何处理大项目上下文不足
- 它什么时候需要用户补充信息

## 第三阶段：工具与权限

目标：实现工具调用边界和最小安全策略。

观察点：

- 什么时候读文件
- 什么时候搜索
- 什么时候执行 shell
- 什么时候请求确认
- 如何避免破坏用户改动

## 第四阶段：OpenCAI MVP

目标：实现一个极简 OpenCAI CLI Coding Agent。

最小能力：

- 读取项目规则
- 搜索和读取文件
- 执行命令
- 生成补丁
- 运行验证
- 记录任务过程

## 当前原则

- 先明确边界，再实现
- 先闭环，再扩展
- 只保留能帮助理解或实现的材料
